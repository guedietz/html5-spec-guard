# Display & Video 360 (DV360) — HTML5 Banner Specifications

Reference for automated compliance checks. Each check has a unique ID for traceability in reports.

---

## Package Requirements

### DV360-PKG-01: Archive Format
- **Allowed formats:** `.zip`
- **Severity:** FAIL

### DV360-PKG-02: Maximum File Count
- **Limit:** 100 files per archive
- **Severity:** FAIL

### DV360-PKG-03: Maximum Package Size
- **Limit:** 5 MB (5,242,880 bytes)
- **Severity:** FAIL

### DV360-PKG-04: Filename Length
- **Limit:** Each filename (including path within archive) must be ≤ 200 characters (≤ 50 for audio/video files)
- **Severity:** FAIL

### DV360-PKG-05: Primary HTML File
- **Rule:** Archive must contain at least one `.html` or `.htm` file at the root level
- **Severity:** FAIL (no HTML found, or HTML in deep nested path) /
  WARNING (HTML inside a single wrapper folder — macOS Archive Utility pattern)

## File Requirements

### DV360-FILE-01: Allowed File Types
- **Allowed extensions:** `.html`, `.htm`, `.css`, `.js`, `.jpg`, `.jpeg`, `.gif`, `.png`, `.svg`, `.dfont`, `.eot`, `.otf`, `.tte`, `.ttf`, `.woff`, `.woff2`
- **Severity:** FAIL
- **Note:** Files without extensions or with disallowed extensions must be flagged

### DV360-FILE-02: No Nested ZIP Archives
- **Rule:** Archive must not contain nested `.zip` files
- **Severity:** FAIL

---

## HTML Requirements

### DV360-HTML-01: Ad Size Meta Tag
- **Rule:** Primary HTML file must contain a `<meta>` tag declaring the ad dimensions
- **Pattern:** `<meta name="ad.size" content="width=X,height=Y">`
- **Validation:** `X` and `Y` must be positive integers
- **Severity:** FAIL

### DV360-HTML-02: Valid HTML Structure
- **Rule:** Primary HTML file must contain `<!DOCTYPE html>`, `<html>`, `<head>`, and `<body>` tags
- **Severity:** WARNING

## Click Tracking — Display Ads

### DV360-CLICK-01: clickTag Variable Present (Display)
- **Rule:** For standard display ads (no Enabler.js), the JavaScript code must define a `clickTag` variable. Also detects multi-clickTag variants (`clickTag1`, `clickTag2`, etc.) and lists all exit variables found.
- **Patterns to search:**
  - `var clickTag` / `var clickTag1` / `var clickTag2` …
  - `let clickTag` / `let clickTag1` …
  - `const clickTag` / `const clickTag1` …
  - `window.clickTag` / `window.clickTag1` …
- **Applies when:** No reference to `Enabler.js` or `Enabler` API found
- **Severity:** FAIL

### DV360-CLICK-02: clickTag Not Hardcoded (Display)
- **Rule:** The `clickTag` value must not be a hardcoded URL
- **Patterns that trigger WARNING:**
  - `clickTag = "http://..."` or `clickTag = 'http://...'`
  - `clickTag = "https://..."` or `clickTag = 'https://...'`
- **Acceptable patterns:**
  - Assignment from URL parameters
  - Placeholder values
- **Applies when:** No reference to `Enabler.js` found
- **Severity:** WARNING (clickTag hardcoded to a URL) / INFO (clickTag not found — N/A)

## Click Tracking — Rich Media (Enabler)

### DV360-RICH-01: Enabler.js Reference
- **Rule:** Rich media ads must reference `Enabler.js` from the Studio CDN
- **Pattern:** `<script src="https://s0.2mdn.net/ads/studio/Enabler.js">`
- **Detection:** Search for `Enabler.js` in HTML files
- **Severity:** INFO (detection only — determines which click checks apply)

### DV360-RICH-02: Enabler Exit Events
- **Rule:** Rich media ads must use `Enabler.exit()` for click-through. Extracts named exit identifiers from `Enabler.exit("name")` calls and lists them in the check message.
- **Pattern:** `Enabler.exit("...")`
- **Applies when:** `Enabler.js` reference found
- **Severity:** FAIL

### DV360-RICH-03: Enabler Initialization
- **Rule:** Rich media ads should check for Enabler initialization
- **Patterns:**
  - `Enabler.isInitialized()`
  - `studio.events.StudioEvent.INIT`
- **Applies when:** `Enabler.js` reference found
- **Severity:** WARNING

## Animation

### DV360-ANIM-01: Maximum Animation Duration
- **Limit:** 30 seconds
- **Detection heuristics:**
  - Check CSS `animation-duration` and `animation-delay` values
  - Check `setTimeout` / `setInterval` with values > 30000 ms
  - Check GSAP / anime.js timeline durations if present
- **Severity:** WARNING
- **Note:** Heuristic check — cannot guarantee accurate detection of total animation duration

### DV360-ANIM-02: Animation Looping
- **Rule:** Animations should not loop indefinitely
- **Detection:** Check for `animation-iteration-count: infinite` in CSS, or unbounded `setInterval` without clear stop
- **Severity:** WARNING

## Audio & Video

### DV360-AV-01: No Autoplay Audio
- **Rule:** Audio must not autoplay
- **Detection:** Search for `<audio` tags with `autoplay` attribute, or `new Audio()` with immediate `.play()`
- **Severity:** FAIL

### DV360-AV-02: Video Must Be Muted
- **Rule:** If video elements exist, they must include the `muted` attribute
- **Detection:** Search for `<video` tags without `muted` attribute
- **Severity:** FAIL

### DV360-AV-03: Video playsinline Attribute
- **Rule:** If video elements exist, they should include the `playsinline` attribute for iOS inline playback
- **Detection:** Search for `<video` tags without `playsinline` attribute
- **Rationale:** Without `playsinline`, iOS Safari opens video in fullscreen mode, breaking the ad experience
- **Severity:** WARNING

## Polite Loading

### DV360-POLITE-01: Polite Load Threshold
- **Rule:** If total uncompressed asset size exceeds 150 KB (153,600 bytes), the banner should implement polite loading
- **Rationale:** DV360 itself does not enforce an initial load limit, but Google's third-party ad serving requirements and IAB LEAN standards recommend a 150 KB initial load. Publishers and exchanges serving these ads typically enforce this.
- **Detection:** Check for polite load patterns:
  - `window.addEventListener("load", ...)`
  - `document.addEventListener("DOMContentLoaded", ...)`
  - `Enabler.isInitialized()` or `Enabler.isPageLoaded()`
- **Enhanced message:** Reports total size, estimated initial payload (from `<head>` and pre-polite `<body>` references matched to ZIP entries), and polite load detection status
- **Severity:** WARNING
- **Note:** Heuristic check — cannot guarantee correctness

### DV360-POLITE-02: Initial Load Image Size
- **Rule:** If polite loading is needed (total > 150 KB), individual images should be under 40 KB (40,960 bytes) for initial load
- **Rationale:** Keeps the initial payload within the 150 KB budget when heavy assets are deferred
- **Detection:** Look for images referenced before the polite load trigger
- **Severity:** WARNING
- **Note:** Heuristic — may not accurately identify the initial load image

## Security

### DV360-SEC-01: HTTPS Only
- **Rule:** All URLs referenced in HTML, JS, and CSS files must use `https://` protocol
- **Detection:** Search for `http://` URLs (excluding `http://www.w3.org` namespace references)
- **Severity:** FAIL (HTTP URL to an untrusted domain) /
  WARNING (HTTP URL to a known-trusted domain — still should be upgraded to HTTPS)

### DV360-SEC-02: No Local Storage
- **Rule:** Code must not use `localStorage` or `sessionStorage`
- **Patterns to search:** `localStorage`, `sessionStorage`
- **Severity:** FAIL
